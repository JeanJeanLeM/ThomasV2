## 📝 **Conventions Établies**

### **Nommage Fichiers**
- **MAJUSCULES** : Tous les fichiers .md
- **Underscores** : Séparateurs (`FORM_MIGRATION_GUIDE.md`)
- **Préfixes** :
  - `AUDIT_` : Audits
  - `FIX_` : Corrections
  - `GUIDE_` : Guides
  - `QUICK_` : Références rapides
  - `FINAL_` : Versions finales

### **Organisation Dossiers**
1. **testing/** - Tout ce qui concerne les tests
2. **chat/** - Système de chat et agent IA
3. **design/** - UI/UX et design system
4. **forms/** - Formulaires (migration, guides)
5. **deployment/** - Build, stores, publication
6. **troubleshooting/** - Debug et fixes
7. **observations/** - Fixes spécifiques observations
8. **archive/** - Documents historiques ou obsolètes

### **Documents Racine**
- Garder uniquement les docs "core" (architecture, agent IA, systèmes)
- README.md principal à la racine du projet
- INDEX.md dans docs/ pour navigation

## 🔍 **Comment Trouver un Document Maintenant ?**

### **Méthode 1 : Par Thème**
1. Identifier le thème (test, chat, design, etc.)
2. Aller dans `docs/[theme]/`
3. Parcourir les fichiers du dossier

### **Méthode 2 : Via INDEX.md**
1. Ouvrir `docs/INDEX.md`
2. Chercher par thématique ou nom
3. Cliquer sur le lien vers le document

### **Méthode 3 : Recherche**
```bash
# Rechercher dans tous les docs
grep -r "mot-clé" docs/

# Rechercher dans un thème spécifique
grep -r "mot-clé" docs/testing/
```
